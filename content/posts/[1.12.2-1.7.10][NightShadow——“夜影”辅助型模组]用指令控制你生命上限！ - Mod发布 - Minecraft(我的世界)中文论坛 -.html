关于这个模组的名字：
还记得最最最开始的时候，这个模组的名字是“生命与伤害调整Mod”这样，
不过后来这个模组的功能已经远超过了纯粹的、对玩家生命与伤害调整的范畴了，于是楼楼决定使用一个比较伤感的名字为该模组命名（可能是因为楼楼喜欢伤感吧= =）
然后，这个模组就被命名为 “夜之泪” 模组，很长时间以来不论在贴吧发帖还是自己改都用的是这个。
后来，额觉得这个名字太过于不专业了，所以最终还是决定将该模组的名字命名为“夜影”
这样既霸气又伤感对吧
注：本模组虽然名字是“夜影”，不过模组ID仍然是“夜之泪（nighttears）”，因此该模组生成的配置文件名字仍为“nighttears.cfg”，并且在拆解这个模组时也会发现许多“nighttears”这样的单词。
不造大家第一眼看到这俩名字时能不能和玩家的属性联系在一起

emmmm……所有的命令都已经写在上面啦，够详细吧，当然这里我还是要简单介绍下：
玩家拥有的七项属性是：生命上限、基础攻击伤害、击退抗性、移动速度、近战攻击伤害修正系数、远程攻击伤害修正系数、攻击速度（1.10.2以上版本特有）。
分别对应相应的 set 命令和 add 命令，set命令可以直接设置玩家的某项属性，add命令可以在原有属性的基础上对该属性进行增减，比如 +10 生命上限等等，如果是减少的话直接将 add 命令的数值变为负数即可。
其中 /getProperties [player] 这条命令可以直接获取玩家的属性状态：

而 /cleanProperties [player]这条指令可以清空玩家属性，将玩家的属性全部还原为配置文件状态。

关于命令的填写：
首先，上图中标注 [] 的地方都不是必填项，因此如果你不标明玩家的话，这些命令的目标都是你自己。
比较特别的是这个设置药效的 /setPotion [player]药效ID [药效级别]命令，只输入一个参数的话默认药效级别为 1 ，如果只输入两个参数的话则是 药效ID+级别，如果要指明某位玩家请将该命令填写完整。
在输入命令的时候，只要先输入其中一部分首字母然后按下 Tab键 即可将命令自动补充完整。
由于以上所有命令均符合一般命令使用规范，同样，在输入玩家姓名时，可以按 Tab 键自动补充玩家名称。

填写的注意事项:
请勿填写神马神马不合理的设置，比如 /setMaxHealth 某位玩家 0 酱紫，除非你想制裁熊孩子。
同样呢，设置药效的命令ID范围 1.7.10 版本是 1到23, 1.10.2 之后的版本是 1到27，输入其他范围的数字是完全无效的。
当然啦，击退抗性（KnockbackResistance）的数值只能是 0 到 1.0，不可能有其他的范围了。
最后，温馨提示：/setWalkSpeed 命令可以控制玩家速度，切记将该数值调整至 1 或者以上，像是 0.5 就已经很快了，玩家默认的数值不过才 0.1 而已，经常玩 NBT 的小伙伴应该很熟悉这个数字，当然如果你将该数值设置为10，你就能体验到飞出这个世界的快感了。

关于使用的权限：
除了 get 命令外，所有的设置命令只有腐竹、单机模式作弊的玩家还有在命令方块内执行。
当然如果你觉得玩家get别人的属性算是侵犯隐私的话，可以在配置文件中修改get命令的使用权限。
由于这些命令均可以在 命令方块 内执行，所有有哪位大神能利用介个模组做成解谜地图啥的捏？

玩家的初始属性与配置文件
楼楼力争将该模组做成一个即是创造模式中的玩家又是单机模式中的玩家可以使用的模组，尽管非作弊模式玩家不可以通过指令调整自己的生命上限，但可以通过配置文件调整自身的默认生命上限，如果觉得生存太过于困难呢，就调高自己的属性；反之，如果觉得虐怪太轻松呢，就降低下自己的属性来稍稍“惩罚”下自己吧= =。
运行游戏后，生成的 nighttears.cfg 配置文件内容如下：

authorities {# Allow MaxHealth = 0?(default:false)
    B:allow0Health=false//是否允许玩家生命上限被调整为 0 ，慎用！生命上限调整为 0 后将无法重生。如果要在一个已将生命上限调整为 0 的存档重生，请将该配置设置为 false 并重启 Minecraft 然后重新载入存档。
# If the configure initial attributes you set are effective?(default:true)
    B:allowConfigEffect=true//是否采用配置文件关于默认属性的设置，该项决定下面的默认玩家属性配置是否生效，选择否的话则玩家的默认属性永远是原版的默认属性。
# If true,player health will change with maxHealth's change.(default:true)
    B:healthChangeWithMax=true//当生命上限发生改变时，生命值本身是否会随同一起改变。在某些不希望玩家生命上限的变动影响生命值的情况下可以关闭改配置。
# If the command getPlayerProperties and getPotions only can be used byOP?(default:false)
    B:onlyOPCanUseGetCommand=false//类似于 /getPotions、/getProperties 这样的获取性指令是否只允许op使用，在服务器内如果腐竹不希望玩家窥探其他人的属性的话可以将该命令设置为 true 。
}
default attributes {
# Default damage.(default:3)
    D:attackDamage=3.0//默认基础攻击力，注意：当使用 /cleanProperties命令时将会将玩家的属性值全部还原为该默认值。
# Default attackSpeed,1 to 1024.(default:6)
    D:attackSpeed=6.0//默认攻击速度，1.7.10版本无此配置项。
# Default closedamagefix.(default:1.2)
    D:closeDamageFix=1.2//默认近战伤害修正系数， 你的近战攻击伤害 = 你的攻击力 × 该属性。
# Default back resistance of attrack,0 to 1.(default:0.25)
    D:knockbackResistance=0.25//默认击退抗性， 1 表示永远不会被击退。
# Default maxHealth.(default:24)
    D:maxHealth=24.0 //默认最大生命上限，请勿设置为 0 。
# Default remotedamagefix.(default:1.2)
    D:remoteDamageFix=1.2//默认远程伤害修正系数，原理同上。
# Default walkspeed.(default:0.12)
    D:walkSpeed=0.12//默认移动速度，请勿设置过高。
}
player hud{
# If player's maxHealth<100,also show the player's propertiesastexts?(showHealthAndArmorBarAsText must be true,default:false)
    B:showAsText=false//当你的生命上限并未超过 100 ，仍将你的 HUD 中的生命条以文本的形式显示。如果你不喜欢原版的那个红心条的话……其中最下面的那个属性要设置为 true 该属性方可生效。
# Show food and air bar as text when life showing as text?(default:false)
    B:showFoodAndAirAsHealthText=false//当生命值以文本方式显示时，饥饿度的条和氧气条是否也同样以文本方式显示。
# IfmaxHealth > 100,show health and armor bar as text?(default:true)
    B:showHealthAndArmorBarAsText=true//当生命值超过 100 时，是否将生命条以文本的方式显示，如果选否，请注意如果生命上限过高的话你的血条会糊住你的屏幕，如下图：
}
