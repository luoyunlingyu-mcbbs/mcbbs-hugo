———————————钱 币 革 新———————————

【简介】

Numismatic Overhaul为Minecraft增加了一个货币系统，替代了Minecraft原有的货币系统，让你不用为挖不倒绿宝石而感慨奸商。本模组的创作源于Terraria，但是它又与Terraria的做法不太一样，所有请不要把他们混为一谈。如过你也讨厌为什么挖不到绿宝石，只好村民们交易，而且这些村民给的少的伙伴们可以看看这个模组。这个模组有多种货币获取渠道，可以尽量避免与村民们交易和挖矿，如果你感兴趣的话，来尝试一下吧！

【视频简介】

超级八里奥 大大的作品，是1.18.1版本的介绍，视频仅限参考。
【模组简介】

这是一个以货币与贸易为主的模组。本模组添加了多种获取货币的方式尽量避免与奸商们交易和挖矿，也添加了在多人游戏与玩家交易的方式，使玩家成为了"店主"，该模组添加了：货币，商店，储钱罐，钱袋。本模组在不断的更新和完善，我也会不断的更新本贴，给我们的读者一份最新的模组介绍。

=货币=

本模组一共添加了三种货币，分别为：金币，银币，铜币，可以放在背包中。分别对应面值：10000元，100元，1元。

=商店=


商店是为了在多人游戏中可以玩家与玩家之间可以更好的交易，让玩家向其他玩家出售物品的一个功能方块。

=储钱罐=


储钱罐可以储存货币，减少硬币的损失。

=钱袋=


钱袋在背包在背包界面的右上角（在创造模式看不到），可以查看钱袋的余额，也可以把钱取出来。

【模组使用】
1.商店

商店是为了在多人游戏中可以玩家与玩家之间可以更好的交易，让玩家向其他玩家出售物品。

商店的有两个页面，分别对应库存页面（图标为箱子）和交易页面（图标绿宝石）。

在交易页面中，你可以通过右上方的小组件来添加新的交易。
首先把准备要出售的商品拖到右上方的小组件中的方格中（要设置好要出售的数量[1]），然后设置价格，都搞好后点击对号，那么一个交易就创建完成了[2]。

如果你想改变交易的价格，需要点击交易列表中的项目，重新填写价格，并点击对号就完成了修改。如果你想修改交易的价格那只有删除交易重新添加；如果你想删除交易，点击交易列表中的项目，并点击叉号来删除它。

如果想要动添加库存，那可以使用漏斗。首先点击商店右旁的有这漏斗的图标，把“不可以”改为“可以”，这样漏斗就可以使用，使物品自动添加到库存中。

在商店右旁有一个和钱袋一样的一个小组件，这个组件可以显示该商店的收入并且可以提取该商店的收入。单击该组件的箭头会立即将所有硬币提取到钱袋中。


注意：[1]设置好的商品会出现在左侧的商品页中，一个商店一共可以设置24个商品，同一个物品不同的数量也算作一个商品。

[2]出售的物品可以设置1-64个（请根据原版的堆叠来进行，该数值仅作参考）

在多人游玩该该模组时，谨防商店被挖掉！商店被挖掉后，在商店的物品将会掉落！


如果想查看商店中的交易，就像其他玩家看到商店中的交易；需要按住潜行时右键单击商店，这样就可以打开交易界面，这个界面就是其他玩家看到的界面。这个界面可以理解为村民交易时的界面，只是它没有经验条和打折。







如果想要编辑村民交易，你需要按照此页面上的数据格式制作一个数据包。
有一些设置可以在配置文件[3]中找到。

注意：[3]配置文件：

未开启版本隔离：在.minecraft\config文件夹中numismatic-overhaul.json5名字的文件。

开启版本隔离：在.minecraft\versions\你给你游戏版本起的名\config文件夹中numismatic-overhaul.json5名字的文件。


2.死亡掉落

死亡后，默认掉落钱包中硬币总量的10%。

如果要更改此设置，可以在创建世界时更改gamerule moneyDropPercentage这条规则;如果已经创建好了世界，需要使用/gamerule指令。

示例：死亡时掉落50%的硬币：/gamerule moneyDropPercentage 50


3.货币和钱袋

钱袋可以储存硬币，可以把硬币放到主手中，右键就把钱存进了钱袋里了。

在钱袋中，每100个面值地的自动兑换成面值高的，但是在钱袋外面不会。

钱袋既可以查看我们储存在里面的硬币，还可以取出。首先我们打开背包，点击钱袋[4]，会出现钱币栏，我们可以选择要取出硬币的数量，选好后点击钱币栏最下面的箭头就可以取出来了。

硬币自动堆叠时，最高可以堆叠99个；而手动堆叠时可以无限堆叠[5]，并且硬币的图标变成了一个小袋子。

钱袋也可以在与村民交易时使用，前提是你储存了足够的硬币。

如果在安装完其他的模组后出现钱袋消失，可以在配置文件中找到purseX和purseY，修改这两个值即可。如果安装了Mod Menu（模组菜单）则可以直接打开配置面板进行配置。


注意：[4]在创造模式看不到钱袋，如要看到请到配置中修改。

[5]视频中的手动堆叠bug数量为几十亿（未验证）。

推荐安装 Mod Menu（模组菜单），这样可以更好的配置模组。CurseForge  Modrinth


4.村民交易

把原版村民的交易货币从绿宝石换成这款模组的货币，并调整了村民交易的物品，其它性质，如打折，有经验条等性质不变。

在与村民交易时也可以直接使用储存在钱袋的硬币，前提是你储存了足够的硬币。


注意：本模组支持 More Villagers（更多村民）模组。Forge Fabric


5.货币获得途径

a.击杀掠夺者

在本模组中击杀掠夺者会随机掉落9到35个铜币，有几率会掉落一个银币。

如果你想让更多的生物掉落硬币，必须需要将它们添加到这个实体标签中：numismatic-overhaul:the_bourgeoisie

b.战利品箱

从0.2.4版本往后的版本，可以通过配置文件或在模组配置页面中调整：“结构”中的硬币的数量。

以受“结构”选项影响的战利品箱：


  • 堡垒战利品箱
  • 被掩埋的战利品箱
  • 要塞走廊的战利品箱
  • 掠夺者前哨站中战利品箱

注意：“地牢战利品”选项也会影响废弃矿井。

配置文件：

未开启版本隔离：在.minecraft\config文件夹中numismatic-overhaul.json5名字的文件。

开启版本隔离：在.minecraft\versions\你给你游戏版本起的名\config文件夹中numismatic-overhaul.json5名字的文件。



【说明】

本模组只有 Fabric 版本，需要 Fabric API 和 owo-lib 作为前置！

如果 owo-lib 前置没安装则会出现这个界面。

        
      直接点击"Download"然后点"Close"就下好，然后就会进入到游戏里了。
  • 如果你在按住Shift键的同时点击钱袋的按钮，在你背包中的你所有硬币将会立即存入你钱袋中。
  • 如果你在按住Shift键的同时点击钱袋里的箭头，你将可以以10为单位显示在框中，而不是以1为单位。
  • 如果你想提取你在钱袋里的所有硬币，你只需按住Shift+Ctrl键，然后点击钱袋里的箭头，你的硬币就全部取出了。
  • 本模组前置 owo-lib 与 Optifabric 模组有冲突，请注意。（感谢七楼的反馈）

【问题解惑】

1.作者不尝试修复商店可以被其他玩家挖下来的问题吗？

Because allowing people the create infinite unbreakable blocks is just a bad idea in many different ways, this will not be implemented natively in Numismatic. If you need to have protected shops, I recommend you install one of the many claim/protection mods like FTB Chunks or Get Off My Lawn

Cheers


意思大概为：让玩家创建一个不可以摧毁的方块是个不好的注意，所以不会在模组中实现。如果你实在想要保护此方块，请安装mod来进行。


2.可以用绿宝石兑换硬币吗？

There are some trades spread over a few of the villagers which already more or less let you do this. In general we do not plan on adding any sort of direct exchange mechanism - after all the entire point of Numismatic is to move away from Emeralds as a currency. So to answer you question: No, Emeralds do not serve much of any purpose currency-wise

Cheers


意思大概为：作者不打算增加任何形式的直接交换机制，因为Numismatic Overhaul的意义在于创造新的货币，而不是相互转化。所以绿宝石在货币方面没有作用。

本次更新不仅可以使钱包可以在生存中使用，还可以在创造，商店和村民等有关交易的界面中使用。还有，商店现在可以命名了。

2023.7.24 0.2.9
This build removes the MCA compatibility mixin to restore compatibility with newer versions of MCA where tax items can be natively configured
此版本删除了 MCA 的混合插件，用来恢复较新版本的 MCA 兼容性，并在较新版本可以对钱币进行设置。

2023.6.10 0.2.9
This is a straightforward port to 1.20 - support for older versions will be dropped
该版本是对1.12.0版本的直接迁移-我们将放弃对旧版本的支持。

2023.3.27 0.2.9
Changes
变化
The built-in integration with MCA Reborn taxes has been removed. This fixes crashes with newer MCA versions (#78) and is also not required anymore, as this can now be achieved through MCA Reborn's configuration
为了修复最新的 MCA Reborn 模组会导致本模组崩溃，所以删除了 MCA Reborn 该模组的内置税收功能。现在可以通过 MCA Reborn 的配置来实现这个功能。
The purse balance change message that's displayed in the action bar by default can now be moved to chat or disabled entirely (#58)
在物品栏中显示的钱包余额变化信息可以在聊天栏中显示，也可以不显示。
Fixes
修复
The purse no longer uses its default position when the recipe book is opened and correctly respects the config options (#74)
打开物品栏时，钱包的位置并不使用默认的配置位置，而是根据配置选项的位置显示。
The /numismatic balance player get command now actually displays the balance
这个 /numismatic balance player get 命令可以显示玩家钱包中实时的余额。

2022.12.28  0.2.8
Fix dupe when trying to pick up money in a Piggy Bank through a money bag
修复了当试图用钱袋提取存钱罐的货币时重复的问题。
