介绍
这是一个高度自定义物品的插件，可以实现自定义合成配方，自定义材质，自定义物品属性，自定义方块掉落等功能，最重要的是它是免费的！如果用得好的话，完全不逊色于其他同类型的付费的插件。目前尚不支持自定义方块材质！

支持的Minecraft版本
Minecraft 1.8.x和更早版本： 无法支持（至少不要添加mods）
Minecraft 1.9.x，1.10.x和1.11.x：  不受支持，但是有足够的兴趣去尝试
Minecraft 1.12.0和1.12.1：  未经测试，但可能会正常工作
Minecraft 1.12.2： 受支持，但缺少一些功能，例如自定义三叉戟
Minecraft 1.13.0和1.13.1：  不受支持，但是如果有足够的兴趣的话会去尝试
Minecraft 1.13.2：  受支持，但缺少一些功能并且没有经过很好的测试
Minecraft 1.14.x：受支持，但缺少Minecraft 1.15中的功能
Minecraft 1.15.x：  受支持，但尚不能使用在minecraft 1.15中添加自定义物品
Minecraft 1.16.x及更高版本：  在撰写本文时不存在

特色
  • 自定义名字、注释、属性和材质的物品
  • 自定义耐久和伤害的装备
  • 自定义速度、耐久和攻击力的弓
  • 自定义盔甲和耐久的盔甲
  • 自定义方块和生物的掉落
  • 自带一个方便的全功能编辑器！但是UI不忍直视

  • 前置插件原贴地址 https://dev.bukkit.org/projects/knokko-core
  • 兼容性（红色为不兼容，绿色为兼容，褐色为可能导致错误）：
  • Crazy Enchantments
    该插件对自定义物品上的HellForged附魔有很好的支持。如果使用HellForged附魔来修复自定义物品，则将修复其自定义耐久性，而不是原版耐久。如果未安装疯狂附魔，则什么都不会发生，除了会控制台中显示一条消息，指出似乎未安装疯狂附魔。
    Ultra Chest
    该插件进行了很小的更改，允许用户在其配置文件中使用本插件的自定义项目。
    其他可以自定义物品尤其是自定义材质的插件
    与其他自定义物品插件可能会引起问题，尤其是当它们也具有自定义材质时。如果该其他插件使用与该插件相同的自定义材质机制，则会出现怪异的效果。
    Item Fixer
    ItemFixer插件删除了项目的所有“欺诈”属性，例如nbt标签和不可能的附魔。由于所有自定义物品都具有此类“欺诈”属性，因此，一旦任何非op玩家将其移到手中，这将导致所有自定义物品删除其材质和特殊属性
    其他可以更改合成配方的插件
    为防止自定义物品在原版合成配方中使用，此插件将阻止任何尝试使用未在编辑器中注册为自定义合成配方的，使用自定义物品作为合成材料的合成。不幸的是，这也阻止了其他自定义合成配方插件在其配方中使用自定义物品。不过，所有使用原版物品的合成配方都应该仍然有效
  • （注:本内容中所有“自定义物品”均指非原版物品，例如拥有lore）

1.创建自定义材质:
你必须先去官方网站下载插件的编辑器除非你想在一大堆的文件面前动手打出来一大坨的代码......因为编辑器是一个jar文件，所以你也必须正确安装java。打开编辑器会出现这样的一个界面，这个界面下面称为主菜单
（没错真的就长这样.....）

  • New item set为新建一个物品集
  • Edit item set为编辑一个已经创建了的物品集
  • Combine item sets是合并两个物品集为一个新的物品集
  • 左侧三个小按钮中，上面的两个按钮是作者的discord网址，下面的按钮是打开官方帮助

点击New item set来新建一个物品集，会弹出一个新窗口，在filename里面输入你想给物品集命名的名字，然后点create，会出现下图界面。下文中称为物品集编辑菜单

  • save是保存，
  • save and quit是保存并退出到主菜单，
  • export for <版本>是以...版本的格式导出物品集（点击后会回到主菜单，这时候必须点Exit editor退出编辑器，才能生成物品集文件）
  • textures、items、recipes、drops、projectiles分别为创建自定义材质、物品、合成配方、弹射物（存疑，类似于被发射的弹药），点击会打开各自的编辑菜单
  • recipes及其后的教程有待补充

首先我们要新建一个材质，点击textures按钮会出现下图界面(滑稽是我自己创建的)

  • Back为返回，下文不再解释
  • Load texture为新建一个材质
  • 右侧会显示已经创建过的材质的列表，在物品右侧显示的edit为编辑该物品，copy为复制，delete为删除

点击load texture，出现如下界面

本帖暂时只讲述创建普通材质（Load simple texture）

  • name是你的材质的名字，后面会用到
  • edit用来导入你的材质，准备好后点击create创建，选择的图片必须是16 x 16像素，32 x 32、64 x 64、128 x 128、256 x 256或512 x 512的图像，但是很多像素太大的材质会延长您的玩家进入服务器自动下载资源包所需的时间，本插件暂不支持自定义方块材质！所以不要导入方块材质！
比如我创建了一个名为huaji的材质，导入材质并创建后会是这样的

然后保存，点back回到物品集编辑菜单（不保存将会撤销！），我们就可以开始创建自定义材质的物品了

2.创建自定义物品

回到物品集编辑菜单

点击物品集菜单的Items按钮会出现一个新的界面，类似下图

  • Create item为新建一个物品
  • Command block help是用命令方块给予自定义物品的帮助（不明）
  • 右侧会显示已经创建过的物品的列表，作用与材质列表的按钮相同

点击界面中的create item按钮创建物品，在新的物品里面选择要创建的物品类型，本帖只讲述创建普通物品（simple item）。至于其他的选项，因为只要翻译一下就差不多能看懂是什么类型（如sword是剑），所以就不赘述了。弓不能使用一般物品材质（simple texture）！必须使用Bow texture！

点击普通物品（simple item）出现一个新界面

（某吃瓜少女好像乱入了......）
  • name是物品的id，而非物品的显示名
  • 第二三项本贴暂不讲述，请不要修改
  • display name是物品的显示名，可以用颜色代码§和中文
  • lore是注释，
  • attribute modifiers是一些物品属性
  • default enchantments是默认附魔
  • item flags是一些隐藏设置
  • texture是选择材质
  • model本教程暂不讲述
  • on-hit player effects是打中玩家时的效果
  • on-hit target effects是打中任何实体时的效果
  • commands是使用物品时执行的指令
  • max stacksize是最大堆叠数

进行您所需要的编辑后，点击create创建自定义物品（编辑器中的中文无法正常显示，显示为乱码，但是实际上还是能正常使用），比如我创建了一个名为huaji的物品，材质为上述的huaji，

创建完成后保存并按照你的游戏版本导出（直接退出会撤销所有的操作！！！），默认导出的文件位置在<我的文档>的Custom Item Sets文件夹里，即C:\Users\用户名\Documents\Custom Item Sets，我创建的物品集为texture1，所以导出后是下面的效果


3.让插件加载物品集

将<物品集名>.cis文件放入服务器插件文件夹里的CustomItems文件夹（运行一次插件后会自动创建，当然您也可以手动创建），重启服务器或者运行/ci reload（6.3版本以上限定）指令，然后上传你的材质包（即图中的<物品集名>.zip），并在服务器设置服务器材质包（当然你也可以选择让玩家手动下载安装）。


4.上传、设置服务器材质包

我（指作者）推荐使用minepack托管材质包，您可以在此站点免费托管材质包，因为服务器必须始终可以从网络上访问材质包。上传完材质包后你会得到类似下面的网页


然后你的玩家进入服务器的时候，客户端就会自动下载资源包了，记得让您的玩家开启允许使用服务器资源包！输入/customitems give <物品的插件内部id> [玩家]来给予玩家一个自定义物品
比如我按步骤安装了我的texture1物品集之后，输入/customitems give huaji @p 就能给我一个滑稽


5.编辑已创建物品集

在主菜单界面点击Edit item set以查看和编辑你已经创建了的物品集，会出现下图界面鬼知道作者这ui怎么做成这样了.....点击右侧的<物品集名> 按钮就可以打开该物品集的物品集编辑菜单

  • refresh是刷新
  • cancel是取消，即返回主菜单，
  • load back-up会打开备份菜单，类似下图，备份菜单右侧会显示备份的物品集名称和备份时间，每次物品集进行保存，编辑器都会自动备份一次该物品集，您可以在Custom Item Sets / backups文件夹删除任何备份。

加载备份的方法：点击您想要返回到的备份，就会恢复到该备份并打开该物品集物品集编辑菜单，点击所打开的物品集编辑菜单的save按钮才能使恢复生效，否则不会产生任何效果！

备份系统有两个目的：
  • 如果您对最近所做的更改不满意，则允许您加载较旧的版本。
  • 如果由于我（作者）搞砸了而导致最新版本损坏，则允许您加载较旧的版本
