Essentials基础系列插件包括以下七个文件：
1.Essentials.jar　　　　　　　　　　总体设置——支持各种作弊命令，可以设置各种有关服务器功能的插件，以下附属插件的功能都经由该插件实现
2.EssentialsGroupManager.jar　　　 组管理——可以给玩家分类，如普通玩家，VIP，管理员，总管理等
3.EssentialsGroupBridge.jar　　　　 组桥接——支撑上一个插件的功能
4.EssentialsProtect.jar　　　　　　  游戏保护——保护轨道、牌子，还有锁箱功能等
5.EssentialsSpawn.jar　 　 　 　 　出生点设置——设置出生点，仅仅需要一个命令
6.EssentialsChat.jar　　　　　　　　聊天显示设置——可以更改玩家的聊天格式，加前缀后缀尾缀之类，可以设置聊天范围
7.EssentialsGeoIP.jar　　　　　　　玩家来源显示——显示玩家所在的国家，甚至城市
8.VanishNoPickup.jar　　　　　　　限制拾取物品——可以阻止特定玩家拾取物品
这几个是最基本的插件，有的还带有其他的插件，但没什么用，比如XMPP（虚拟E-mail）。

二、EssentialsGroupManager组管理设置

插件加载后对应文件夹位置：plugins\Groupmanager
其下包括worlds文件夹、备份文件夹和config.yml设置文件，后两者可以无视；
worlds文件夹下有各个世界所对应的groups.yml组文件和users.yml玩家文件，是接下来要说明的。

一、groups.yml组文件
文件组设置一般格式（注意空格！）：
groups:                     # 这个要保持在文件最上处
  XXX:                        # XXX的含义是你可以自行增减用户组，名称任意，只要保持各设置格式相同即可
    default: false           # 该项设置玩家第一次进入游戏时，是否自动归入改组，true是，false否，下同
    permissions:           # 该项设置玩家的权限，在下一行之后添加，该项请看完后面的设置再更改
    - essentials.*           # *代表ESS插件某一权限，例如“sethome”为设置家等，如果保持*，该组则拥有Essentials插件的所有权限
    - '*'                        # 仅仅为'*'（注意引号）则是指该组拥有所有的权限（包括上面的）
    - XXX.XXX               # 这是其他插件的权限格式，长短，点数量不定，作者应该会在帖子中作说明，ESS的详细权限在3楼中可以找到（目前未完成）
    inheritance:            # 意为继承，在下一行之后添加其它某一组（级别相邻或相同即可），之后XXX组就拥有所添加的组的所有权限，没有则改为“inheritance: []”，下方组也不添加
    - default                  # 例如该项设置了默认组，则XXX组就拥有默认组所有权限（你不用再在“permissions”中添加命令了）
    info:                       # 用户组属性
      prefix: '&e'            # 前缀，若将引号中的内容改为“&c[VIP]&2”，该组所有玩家名则会显示为：[VIP]绿名，输入“/list”查看在线人数时就可以看到。颜色设置在这里看
      build: true            # 该组是否能修改地形（注：刚安装插件时，默认组此项为false）
      suffix: ''                # 尾缀，好吧，这项我没用过，应该和“prefix”一样功能，不过在昵称尾部

二、users.yml玩家文件
该项设置很简单，一般格式：
users:                        # 该项固定在文件顶部
  123:                        # 我们以一个名叫“123”的玩家举例吧
    subgroups: []         # 子用户组，类似于上面的“继承”，单独对该玩家有效（上面的“继承”同样有效），冒号后接“空格[]”表示“无”，下同
    permissions: []        # 权限，仅该玩家拥有（添加方法同groups.yml），该两项设置一般不需更改
    group: XXX            # 玩家所在用户组

三、EssentialsGeoIP玩家所在地设置

插件加载后对应文件夹位置：plugins\EssentialsGeoIP

一、GeoIP数据库文件
一个是关于国家的，一个是关于城市的（默认只有国家的，需要更改设置）
根据玩家的IP判断玩家所在地的依据，所有的IP地址与其对应的国家和城市的数据保存于此，从某网站下载的。

二、config.yml设置文件
内容&说明如下：
database:                           # 文件开头，不能改动
  show-cities: false               # 是否显示城市，如果更改为true，则会下载一个关于城市的数据库文件，并在玩家登入服务器时显示所在城市，true是开启，false是关闭
  download-if-missing: true  # 如果未找到，是否自动下载
  # IP对应国家的下载地址，无需改动
  download-url: "http://geolite.maxmind.com/download/geoip/database/GeoLiteCountry/GeoIP.dat.gz"
  # IP对应城市的下载地址，无需改动
  download-url-city: "http://geolite.maxmind.com/download/geoip/database/GeoLiteCity.dat.gz"

show-on-login: true            # 当玩家登入服务器时是否显示
show-on-whois: true           # 当输入“/whois 玩家昵称”时是否显示所在地址
