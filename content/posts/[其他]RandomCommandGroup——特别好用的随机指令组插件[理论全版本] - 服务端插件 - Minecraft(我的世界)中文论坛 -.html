RandomCommandGroup
                         - 简单的随机指令组
Github开源
策划服务器的时候有个朋友提出了这个想法
然后我就给他写出来了

简单又不简单的随机指令组
多种增加概率的方式
超高的自定义

功能没什么好说的,概率执行指令
可以通过特殊权限/物品/玩家名来增加概率

help指令 [指令可用tab提示]


run指令


配置文件
  1. #具体的指令
  2. #可无限增加
  3. CommandGroup:
  4.   #指令组id
  5.   #例子
  6.   Example:
  7.     #几率
  8.     Chance: 50
  9.     #权限
  10.     Permission: 'randomcommandgroup.run.example'
  11.     #执行的指令
  12.     # op 以op身份运行
  13.     # console 以控制台身份运行
  14.     # player 以玩家身份运行
  15.     #可以使用延时
  16.     #例子:
  17.     #[op]heal <player>:20
  18.     # 20 代表着延时 1s
  19.     Commands:
  20.       - '[op]heal <player>'
  21.     #成功时发送的信息 [不要可以直接删掉]
  22.     #消息设置与 FlameCore 一致
  23.     Success:
  24.       - ''
  25.     #失败时发送的信息 其余同上
  26.     Fail:
  27.       - ''

  29. #特殊设置
  30. Special:
  31.   #是否启用 [true 是|false 否]
  32.   Enable: false
  33.   #检测列表
  34.   #格式
  35.   #检测模式 词条 几率
  36.   #模式有
  37.   # permission 权限
  38.   # name 物品名字
  39.   # lore 物品lore
  40.   List:
  41.     - 'permission<->rcg.special.VIP<->3'
  42.     - 'name<->VIP专用道具<->1'
  43.     - 'lore<->很牛逼的东西<->3'

  45. #消息设置与 FlameCore 一致
  46. #帮助命令
  47. Help:
  48.   - '&7| &aRandomCommandGroup &7| &e插件帮助'
  49.   - '&f/randomcommandgroup help &a插件主命令'
  50.   - '&f/randomcommandgroup run [玩家] [指令组id] &a为指定玩家运行一个指令组'
  51.   - '&f/randomcommandgroup reload &a重载插件'
  52.   - '&4PS&f: '
  53.   - '&e所有的 &brandomcommandgroup &e都可用 &arcg &e来代替'
  54.   - '&a[玩家] &e不填默认为&7指令使用者'
  55. #没有权限时发送的消息
  56. NoPermission:
  57.   - '[message]&c你没有权利这么做'
  58. #重载插件时发送的消息
  59. Reload:
  60.   - '[message]&a插件重载成功'
  61. #控制台不可执行时发送的消息
  62. CantConsole:
  63.   - '[message]&c后台无法执行该指令'
  64. #参数错误时的消息
  65. ArgsNoEnough:
  66.   - '[message]&c指令参数错误,请检查是否输错了指令'
  67. #配置文件出错时发送的消息
  68. FileError:
  69.   - '&7[&b系统&7]&c配置文件错误,请检查'
