AntiAntiAFK

0x01 前言

我是某个服务器的管理员，最近隐身私访发现服务器有玩家修建了巨型全自动仙人掌田，并且使用类似脚本精灵的软件来实现绕过CMI等基础插件的自动afk功能，我称这种行为为AntiAFK，所以我的插件取名叫AntiAntiAFK(雾)。本插件一开始只有验证码功能，但是在我的服务器试用一天以后，发现有的玩家通过脚本精灵甚至国外某知名作弊mod进行不断重连来实现挂机的功能，于是我临时赶工又添加了玩家被踢次数过多触发临时封禁的功能。我在论坛上找到了功能类似的插件，但是好像作者没有继续维护(作者被封禁了?)，所以我决定自己来重写一个。


0x02 功能介绍

  • 生成随机验证码要求玩家输入
  • 玩家被踢次数过多进行临时封禁

0x03 特性展示

  • 面向玩家的所有文本均可自定义


    • 配置文件

    • 验证码信息

    • 封禁信息


  • 验证码的长度，构成字符可在提供的范围内选择


    • 可以设定长度，在大写字符集、小写字符集、数字集之中进行任意选择


  • 可以使用PlugMan这类插件进行热装载，会自动加载在线玩家，无需重启服务器

  • 自定义验证码出现间隔，玩家出现验证码后剩余的输入时间
  • 大多数程序逻辑使用异步执行，尽量降低服务器开销

  1. # 向玩家显示的信息
  2. Message:

  8.   # 玩家被踢出显示的信息

  11.   # 玩家正确输入验证码提示的信息

  14.   # 玩家错误输入验证码提示的信息
  15.   IncorrectInput: ' &b反挂机 &7> &c验证码输入错误，请重新输入！'

  17.   # 玩家被临时封禁的理由
  18.   # %unban_time% - 临时封禁解除的时间

  22. Title:
  23.   # 主title，即title上面的部分
  24.   Title: '&4请输入验证码'

  28.   SubTitle: '验证码: %verification_code%'

  42. # 单位均为秒
  43. Time:
  44.   # 两次验证码之间的间隔
  45.   Cooldown: 600

  47.   # 从出现验证码到玩家被踢出的时间

  50.   # 插件检测时间，请不要设置的过长或过短(建议就是保持不变)

  54. # 验证码生成设置，不建议太复杂，否则可能劝退玩家(
  55. # 如果所有的都没有打开则默认生成纯数字验证码

  57.   # 验证码长度
  58.   Length: 4

  60.   # 包含数字
  61.   Digit: true

  66.   # 包含小写字母
  67.   Lowercase: false

  69. # 玩家使用脚本重复进入服务器以实现挂机功能？
  70. # 用这个来治他！
  71. TempBan:
  72.   # 是否启动该功能
  73.   Enable: false

  75.   # 只在晚上(零点至次日六点)启动该功能
  76.   EnableOnlyNight: true

  78.   # 玩家连续被踢出几次就会被临时封禁?
  79.   TimesOfKickBeforeBan: 3

0x07 后记

这应该是我写的第二个插件了，第一个写的太丢人了不敢放出来233，经过服务器玩家的反馈，我自己总结出的最佳的配置方案是：

  • 设置验证码位数为1到2位
  • 缩短输入验证码所需的时间
  • 设定验证码生成的范围为数字加小写字母

一开始是打算把这玩意扔到GitHub上面开源的，但是GitHub这两天因为某些原因不是很上的去，我的代理也裂开了，所以，emmmm

关于版本问题，本插件没有使用奇怪的API，也没有使用物品ID，所以理论上兼容性会更广一点，仅在Spigot 1.12.2 | Spigot 1.15.2 | Paper 1.15.2实测，如果其他版本不可用请与我反馈

说句实话这个验证码是有一点点烦人，但是我想了一下，不通过这种方法，监听玩家视角改变、监听玩家移动都会被脚本精灵给欺骗，甚至国外某知名作弊mod还能够以随机路径移动，这使得记录玩家路径并比对这种方式都不再可行，(当然用更高端的算法进行行为分析，写一个对抗算法之类的233，不过效率如何就不好说了)
